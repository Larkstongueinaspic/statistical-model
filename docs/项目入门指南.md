## 1. 项目是什么

### 一句话概括

研究 **2018 年以来美国对华出口管制政策** 是否改变了 **中国从美国进口半导体相关产品的贸易格局**。

### 背景

2018 年起，美国逐步升级对华半导体出口管制——从实体清单扩容到芯片法案，限制范围从制造设备扩大到处理器、存储器等集成电路产品。这个项目用真实的海关贸易数据，量化这些政策对中国进口来源结构的影响。

### 当前结论（v0.2）

```
描述事实：美国份额在多数半导体产品中下降
因果推断：简化模型不支持"美国出口额被显著压低"的强结论
稳妥写法：来源结构重组 + 美国份额承压（不夸大因果效应）
```

### 项目阶段

| 版本 | 状态 | 内容 |
|------|------|------|
| v0.1 | 已完成 | 单产品 `848620`（半导体制造设备），单政策节点 `Post2018` |
| v0.2 | 已完成（当前主线） | 4 产品，`Post2018/Post2022/Post2023` 三节点，完整稳健性 |
| v0.3+ | 规划中 | 事件研究图、更细对照组、可能引入 GDELT 新闻数据 |

## 2. Python 速成：从 C++ 视角

> 如果你已经熟悉 Python 基础语法，跳过本节直接看第 3 节。

### 2.1 解释型语言：没有编译步骤

```
C++:    main.cpp → g++ → a.out → ./a.out
Python: main.py → python main.py  （一次性完成）
```

Python 解释器逐行读取并执行 `.py` 文件。没有编译、没有链接、没有头文件。代价是运行速度比 C++ 慢 10-100 倍，但对于数据分析（瓶颈在 I/O 和矩阵运算）影响不大，因为底层库（NumPy、pandas）是用 C 写的。

### 2.2 `import` ≈ `#include`，但有本质区别

```cpp
// C++: 预处理器把头文件内容"粘贴"到当前文件
#include <vector>
#include "my_class.h"
```

```python
# Python: import 是一个运行时操作，执行目标模块并返回一个模块对象
import pandas            # 导入整个库，使用时写 pandas.DataFrame()
from pathlib import Path # 只导入一个类，使用时直接写 Path()
import numpy as np       # 导入并起别名
```

**关键区别**：Python 没有头文件/源文件分离。每个 `.py` 文件既是声明也是实现。`import` 在运行时执行目标文件的全部顶层代码，然后将其命名空间暴露给调用方。

### 2.3 不需要声明类型

```cpp
// C++: 必须声明类型
int count = 0;
std::vector<double> values;
std::unordered_map<std::string, int> lookup;
```

```python
# Python: 变量只是"标签"，贴在对象上
count = 0          # int
count = "hello"    # 现在变成 str 了，完全合法
values = [1.0, 2.0] # list，类似 std::vector 但可以混装不同类型
lookup = {"a": 1}   # dict，类似 std::unordered_map
```

**但本项目使用了类型注解（type hints）**：

```python
def foo(x: int, y: str) -> pd.DataFrame:
    ...
```

这些注解**不会被 Python 运行时强制执行**——你把 `str` 传给 `x: int`，程序照样跑。它们的作用是给 IDE 和静态检查工具（mypy）看的，类似美化后的注释。习惯写 C++ 的话，建议跟着写，有助于理解函数签名。

### 2.4 `dataclass` ≈ C++ `struct`（但功能更强）

```cpp
// C++: 简单的配置结构体
struct Config {
    const int china_code = 156;
    const std::string product_code = "848620";
};
```

```python
from dataclasses import dataclass, field

@dataclass(frozen=True)  # frozen=True ≈ 所有成员都是 const
class AnalysisConfig:
    china_code: int = 156
    product_code: str = "848620"
    computed_path: Path = field(init=False)  # 不在构造函数参数里
```

`@dataclass` 是一个**装饰器（decorator）**——自动生成 `__init__`（构造函数）、`__repr__`（打印输出）、`__eq__`（比较）等方法。`frozen=True` 使得实例创建后不可修改，类似 `const` 对象。

`field(init=False)` 表示这个字段**不出现在构造函数的参数列表中**，在 `__post_init__` 中计算赋值——类似于在 C++ 构造函数体里计算的成员变量。

### 2.5 `pd.DataFrame` ≈ 超级加强版的二维表

**这是整个项目最重要的数据结构。** 如果你只记住一个概念，记这个。

```cpp
// C++: 二维数据通常这样存
struct Row {
    string product_code;
    int year;
    double import_value;
};
vector<Row> data; // 遍历、过滤、分组、聚合都得手写
```

```python
import pandas as pd

# DataFrame 可以理解为一个"带列名的二维矩阵"
# 每一列有名字和类型，行有索引
df = pd.DataFrame({
    "product_code": ["848620", "854231"],
    "year": [2017, 2017],
    "import_value": [12345.0, 67890.0],
})

# 然后可以用链式调用做各种操作：
df.groupby("product_code")["import_value"].sum()     # 按产品分组求和
df.loc[df["year"] == 2024]                            # 筛出 2024 年的行
df.merge(other_df, on="product_code", how="left")     # SQL JOIN
```

C++ 中要写几十行的筛选/分组/聚合操作，pandas 一行搞定。代价是要适应它的 API 风格（大量链式调用、`axis` 参数等）。

### 2.6 `def` 和函数签名

```python
# Python 函数签名
def build_panel(
    data: pd.DataFrame,           # 参数: 类型注解
    products: tuple[str, ...],    # 不定长元组，类似 std::tuple<std::string, ...>
    config: AnalysisConfig,       # 自定义类型
) -> pd.DataFrame:                # 返回类型注解
    ...
```

与 C++ 的主要区别：
- 参数没有 `const &` vs `&&` 的区别——Python 全是"传引用"，不可变对象天然安全
- 不支持函数重载（一个函数名只有一份定义）
- 返回值可以隐式是 `None`（类似 `void`），不需要显式声明

### 2.7 `lambda` ≈ 匿名函数 / function pointer

```cpp
// C++: lambda
auto func = [](int x) { return x * 2; };
```

```python
# Python: lambda 只能写单行表达式
func = lambda x: x * 2
```

本项目大量使用 lambda 作为临时函数传给 pandas 的聚合操作：

```python
metrics = [
    ("Observations", lambda df: len(df), "{:,.0f}"),
    #              ^^^^^^^^^^^^^^^^^^^^
    #              匿名函数：接收 df，返回 df 的行数
]
```

### 2.8 f-string ≈ `fmt::format` / `std::format`

```cpp
// C++20
std::cout << std::format("Value: {:.2f}", 3.14159);
```

```python
# Python 3.6+ f-string
print(f"Value: {3.14159:.2f}")  # "Value: 3.14"
name = "test"
print(f"File: {name}.csv")      # "File: test.csv"
```

### 2.9 `if __name__ == "__main__"`

这大概是每个 Python 文件里最常见的模式：

```python
if __name__ == "__main__":
    main()
```

它可以这样理解：

- 每个 Python 文件都有一个隐藏的全局变量 `__name__`
- 当你**直接运行**一个文件（如 `python run.py`），这个文件的 `__name__` 被设为 `"__main__"`
- 当你**import** 一个文件（如 `from v02_analysis.runner import main`），被 import 的文件的 `__name__` 是 `"v02_analysis.runner"`

所以这个 if 语句的意思是：**"如果我是主程序入口，就执行 main()；如果我只是被其他文件 import，就什么也不做"**。

这很像 C++ 中 `.cpp` 文件里带 `main()` 和不带 `main()` 的区别——但 Python 没有链接阶段，所以用运行时检查来模拟。

### 2.10 其他关键区别速查表

| 概念 | C++ | Python |
|------|-----|--------|
| 构造列表 | `vector<int> v{1,2,3}` | `[1, 2, 3]` |
| 字典/map | `map<string,int>` | `{"a": 1, "b": 2}` |
| 元组 | `tuple<int,string>` | `(1, "hello")` |
| 条件表达式 | `x ? a : b` | `a if x else b` |
| 布尔值 | `true`/`false` | `True`/`False` |
| 空值 | `nullptr`/`NULL` | `None` |
| 逻辑运算符 | `&&` `\|\|` `!` | `and` `or` `not` |
| 取反 | `!x` | `not x` |
| 整除 | `/`（整数时） | `//`（`/` 总是浮点除法） |
| 方法调用 | `obj->method()` | `obj.method()` |
| 命名空间 | `std::vector` | `pandas.DataFrame` |
| 泛型 | `template<T>` | 不需要（动态类型） |
| 私有成员 | `private:` | `_single_underscore`（约定，不强制） |
| 常量 | `const` | 约定用全大写 `MAX_SIZE = 100` |

## 4. 逐文件走读

### 4.1 入口脚本：`run_v02_analysis.py`

```python
#!/usr/bin/env python3
# ↑ shebang：Unix 系统下执行 ./run_v02_analysis.py 时自动用 python3 解释

"""v0.2 主入口：多产品、分阶段政策节点和补充识别展示。"""
# ↑ docstring：Python 的"文档注释"，放在文件/函数/类的第一行
#   用三引号括起来，可以通过 help() 或 __doc__ 获取

from v02_analysis.runner import main
# ↑ 从 v02_analysis 包的 runner 模块中导入 main 函数
#   类似于 C++ 的 using v02_analysis::runner::main;

if __name__ == "__main__":
    raise SystemExit(main())
#   ↑ 如果是直接运行这个文件：
#     1. 调用 main() 得到返回码（int）
#     2. SystemExit(return_code) 相当于 C++ 的 exit(return_code)
#     3. 如果被 import，这段代码不执行
```

**为什么入口脚本只有 3 行有效代码？**

这是一种常见模式：入口脚本只做"找到 main 函数并运行"，所有逻辑都在 runner 模块里。好处是：
- 可以直接 `python scripts/run_v02_analysis.py`
- 也可以 `python -c "from v02_analysis.runner import main; main()"`（从交互式环境运行）
- 入口脚本不需要包含任何可测试逻辑

### 4.2 配置中心：`config.py`

这是全项目最简单的文件，但决定了所有路径和参数从哪来。

```python
from __future__ import annotations
# ↑ 启用"延迟类型注解求值"——所有类型注解以字符串形式存储
#   好处：避免循环引用问题，同时稍微加快 import 速度
#   从 Python 3.11+ 开始这已是默认行为，但显式写出来保证兼容性

from dataclasses import dataclass, field
from pathlib import Path
# pathlib.Path：Python 3.4+ 的面向对象路径库
#  Path("/a/b") / "c" → PosixPath('/a/b/c')   ← 用 / 拼接路径
#  对比 C++: std::filesystem::path

PACKAGE_ROOT = Path(__file__).resolve().parents[2]
# 逐步拆解：
#   __file__             → 当前文件的路径（相对或绝对）
#   .resolve()           → 转为绝对路径
#   .parents[2]          → 向上两级目录
#   scripts/v02_analysis/config.py → parents[0] = v02_analysis/
#                                  → parents[1] = scripts/
#                                  → parents[2] = 项目根目录/
# 这是 Python 项目中确定项目根目录的常用技巧

@dataclass(frozen=True)
class AnalysisConfig:
    """Central configuration for the v0.2 reproducible workflow."""

    # ── 以下字段可以在构造时传入（都有默认值）──
    root: Path = PACKAGE_ROOT
    candidate_product_codes: tuple[str, ...] = ("848620", "854231", "854232", "854239")
    # tuple[str, ...]：不定长字符串元组。Python 的 tuple 类似 C++ const tuple，不可修改

    china_code: int = 156       # BACI 数据库里"中国"的国家编号
    usa_code: int = 842         # BACI 数据库里"美国"的国家编号
    years: tuple[int, ...] = tuple(range(2008, 2025))
    # range(2008, 2025) → 生成 2008 到 2024 的整数序列（2025 不包含）
    # tuple(...) 把它转成元组，这样也是不可变的

    base_year: int = 2017       # 基准年，用于"份额指数"图
    chunk_size: int = 1_000_000 # pandas 分块读取时每块的行数（1_000_000 = 100 万）
    yearly_file_template: str = "BACI_HS07_Y{year}_V202601.csv"
    # {year} 占位符，后面用 .format(year=2017) 替换

    # ── 以下字段有 field(init=False)，不会出现在构造函数的参数里 ──
    data_dir: Path = field(init=False)
    output_dir: Path = field(init=False)
    data_output_dir: Path = field(init=False)
    figure_output_dir: Path = field(init=False)
    table_output_dir: Path = field(init=False)
    docs_output_dir: Path = field(init=False)
    mpl_config_dir: Path = field(init=False)

    def __post_init__(self) -> None:
        # __post_init__ 在 __init__（自动生成的构造函数）执行完后自动调用
        # 因为 dataclass 是 frozen=True，不能直接 self.x = y
        # 必须用 object.__setattr__(self, "x", y) 绕过 frozen 限制
        # 这是 Python 的一个"后门"——直接操作对象的底层属性字典

        object.__setattr__(self, "data_dir", self.root / "BACI_HS07_V202601")
        object.__setattr__(self, "output_dir", self.root / "results" / "v02")
        object.__setattr__(self, "data_output_dir", self.output_dir / "data")
        object.__setattr__(self, "figure_output_dir", self.output_dir / "figures")
        object.__setattr__(self, "table_output_dir", self.output_dir / "tables")
        object.__setattr__(self, "docs_output_dir", self.root / "docs" / "output")
        object.__setattr__(self, "mpl_config_dir", self.root / ".mplconfig")

    def yearly_file(self, year: int) -> Path:
        # 根据年份返回对应的原始数据文件路径
        return self.data_dir / self.yearly_file_template.format(year=year)
        # "BACI_HS07_Y{year}_V202601.csv".format(year=2017)
        # → "BACI_HS07_Y2017_V202601.csv"

def get_config() -> AnalysisConfig:
    """提供统一配置入口，其他模块不需要重复写路径。"""
    return AnalysisConfig()
```

**为什么用 `frozen=True`？**

防止在任何地方意外修改配置。`config.output_dir = "xxx"` 会直接抛异常。这类似 C++ 的 const 正确性——但 Python 不提供编译器检查，所以用运行时冻结来弥补。

**为什么用 `object.__setattr__` 而不是直接赋值？**

因为 `frozen=True` 的 dataclass 禁止在 `__init__` 之外的地方赋值。`__post_init__` 是在 `__init__` 之后执行的，所以必须绕开这个限制。这是 Python 的一个常见 idiom。

**为什么用 `.copy()`？**

```python
def load_yearly_candidate_trades(year: int, config: AnalysisConfig) -> pd.DataFrame:
    frames: list[pd.DataFrame] = []
    candidate_set = set(config.candidate_product_codes)
    # set ≈ C++ std::unordered_set
    # 用于 O(1) 查找

    for chunk in pd.read_csv(
        config.yearly_file(year),
        usecols=["t", "i", "j", "k", "v", "q"],
        # ↑ 只读 6 列（原始 CSV 有很多列），大幅节省内存
        dtype={
            "t": "int16",   # 年份用 16 位 int 就够了
            "i": "int32",   # 出口国代码
            "j": "int32",   # 进口国代码
            "k": "string",  # 产品 HS6 编码（用 string 因为它是编码不是数值）
            "v": "float64", # 贸易额（千美元）
            "q": "float64", # 数量（吨）
        },
        na_values=[""],
        keep_default_na=True,
        chunksize=config.chunk_size,  # = 1_000_000，每次读 100 万行
    ):
        # ★ 这是 pandas 的 "分块读取" 模式
        # read_csv 返回的不是 DataFrame，而是一个迭代器（iterator）
        # 每次迭代 yield 一个 chunk_size 行的 DataFrame
        # 类似 C++ 中读大文件时分批读到 buffer 里

        filtered = chunk.loc[
            (chunk["j"] == config.china_code) &
            (chunk["k"].isin(candidate_set))
        ].copy()
        # 过滤条件：
        #   j == 156 → 进口国是中国
        #   k ∈ {"848620", "854231", "854232", "854239"} → 只关注这 4 个产品
        # & 而不是 and：pandas 的逐元素逻辑运算必须用位运算符 & | ~
        #                 因为 DataFrame 列是数组，and 只能用于单个 bool

        if not filtered.empty:
            frames.append(filtered)
            # 相当于 C++ vector::push_back

    # 所有 chunk 处理完后的收尾工作
    if not frames:
        # 如果这一年没有任何匹配的记录（理论上不应该发生）
        return pd.DataFrame(
            columns=[
                "year", "exporter_code", "importer_code",
                "product_code", "import_value_kusd", "quantity_tons",
            ]
        )
        # 返回一个"列名正确但空行"的 DataFrame，保证接口一致

    return (
        pd.concat(frames, ignore_index=True)
        # 把所有 chunk 拼成一个完整 DataFrame
        # ignore_index=True：重新编号（否则每个 chunk 自带从 0 开始的索引）

        .rename(
            columns={
                "t": "year",
                "i": "exporter_code",
                "j": "importer_code",
                "k": "product_code",
                "v": "import_value_kusd",
                "q": "quantity_tons",
            }
        )
        # 把 BACI 的单字母列名改为语义化列名
    )
```

**为什么分块读取而不是一次读完？**

BACI 每年的原始 CSV 大概有几十 GB。一次性读入会把内存撑爆。`chunksize=1_000_000` 让 pandas 每次只解析 100 万行到一个临时 DataFrame，用完就丢掉（Python 的垃圾回收会释放内存）。这跟 C++ 中用固定大小 buffer 分批处理大文件是同一个思路。

```python
        .rename(columns={"import_value_kusd": "us_import_kusd"})
```

**什么是"平衡面板"（Balanced Panel）？**

在计量经济学中，"面板数据"指同时具有**横截面**（出口国 × 产品）和**时间序列**（年份）两个维度的数据。"平衡"意味着每个横截面单元在每一个时间点上都有观测值——即使实际贸易额为 0。

```python
    selected_trades = positive_trades.loc[
        positive_trades["product_code"].isin(selected_products)
    ].copy()
    # 只保留被选中的产品

    # 构造"所有可能的组合"——这是核心
    index_frames: list[pd.DataFrame] = []
    for product_code, product_group in selected_trades.groupby("product_code"):
        exporters = sorted(
            product_group["exporter_code"].astype(int).unique().tolist()
        )
        # 对每个产品，生成 出口国 × 年份 的笛卡尔积
        product_index = pd.MultiIndex.from_product(
            [[product_code], exporters, config.years],
            names=["product_code", "exporter_code", "year"],
        ).to_frame(index=False)
        # from_product() 做笛卡尔积：
        #   ["848620"] × [842, 392, ...] × [2008, 2009, ..., 2024]
        #   每个合法组合生成一行

        index_frames.append(product_index)

    panel_index = pd.concat(index_frames, ignore_index=True)
```

**举个具体例子**：假设 `848620` 有 50 个出口国曾向中国出口过。那么生成的面板包含 50 × 17 = 850 行，即使其中很多行对应的实际贸易额是 0。

```python

        ["product_code", "exporter_code", "year"], as_index=False

    panel = (
        panel_index
        .merge(exporter_lookup, on="exporter_code", how="left")
        .merge(product_lookup.reset_index(), on="product_code", how="left")
        .merge(annual_values, on=["product_code", "exporter_code", "year"], how="left")
    )

    # ★ 缺失的贸易额填 0——这是"平衡面板"的关键操作
    panel["import_value_kusd"] = panel["import_value_kusd"].fillna(0.0)
    panel["quantity_tons"] = panel["quantity_tons"].fillna(0.0)
```

**为什么缺失值填 0 而不是删掉？**

如果一个出口国在某一年没有向中国出口某产品，这本身就是一个有信息量的信号（尤其是对于美国——如果美国在某产品上完全没有出口，说明管制很严）。如果把缺失值丢掉，回归会只看"有贸易的"样本，产生选择偏差。

接下来是**特征工程**——创建回归所需的变量：

```python
    # 二进制指示变量（dummy variables）
    panel["US"] = (panel["exporter_code"] == config.usa_code).astype(int)
    # 类似 C++: int US = (exporter_code == 842) ? 1 : 0;

    panel["Post2018"] = (panel["year"] >= 2018).astype(int)
    panel["Post2022"] = (panel["year"] >= 2022).astype(int)
    panel["Post2023"] = (panel["year"] >= 2023).astype(int)

    # ★ 交互项（interaction terms）—— 核心解释变量
    panel["US_Post2018"] = panel["US"] * panel["Post2018"]
    panel["US_Post2022"] = panel["US"] * panel["Post2022"]
    panel["US_Post2023"] = panel["US"] * panel["Post2023"]
    # 这些列 = 1 当且仅当 出口国是美国 AND 年份在政策节点之后
    # 这是 DID（双重差分）的核心构造

    # 互斥阶段变量（用来检查不同时期的边际效果）
    panel["Period_2018_2021"] = (
        (panel["year"] >= 2018) & (panel["year"] <= 2021)
    ).astype(int)
    panel["Period_2022"] = (panel["year"] == 2022).astype(int)
    panel["Period_2023_2024"] = (panel["year"] >= 2023).astype(int)
    panel["US_Period_2018_2021"] = panel["US"] * panel["Period_2018_2021"]
    panel["US_Period_2022"] = panel["US"] * panel["Period_2022"]
    panel["US_Period_2023_2024"] = panel["US"] * panel["Period_2023_2024"]

    # 因变量（被解释变量）的不同变换
    panel["ln_import_value"] = np.log(panel["import_value_kusd"] + 1.0)
    # ln(value+1)：因为 log(0) 是 -∞，加 1 来避免
    # 这是处理零贸易值的常见方法

    panel["asinh_import_value"] = np.arcsinh(panel["import_value_kusd"])
    # arcsinh ≈ 反双曲正弦：在零点附近近似于 ln(x+1)，但数学性质更好
    # 用于稳健性检查

    # 份额型因变量
    totals = panel.groupby(["product_code", "year"], as_index=False)[
        "import_value_kusd"
    ].sum().rename(columns={"import_value_kusd": "product_year_total_kusd"})
    panel = panel.merge(totals, on=["product_code", "year"], how="left")
    panel["import_share"] = np.where(
        panel["product_year_total_kusd"] > 0,
        panel["import_value_kusd"] / panel["product_year_total_kusd"],
        0.0,
    )
    # import_share = 该国该产品出口额 / 中国该产品总进口额

    return panel.sort_values(
        ["product_code", "exporter_code", "year"], ignore_index=True
    )
```

**DID 交互项的逻辑**：

```
US_Post2018 = US × Post2018

对美国 2019 年的观测：US=1, Post2018=1 → US_Post2018 = 1
对日本 2019 年的观测：US=0, Post2018=1 → US_Post2018 = 0
对美国 2015 年的观测：US=1, Post2018=0 → US_Post2018 = 0

回归系数 β 的含义：
"在 2018 年之后，美国相比其他国家的出口变化"
这是双重差分的核心识别逻辑
```

#### 公式语法

```python
STAGE_TERMS = ["US_Post2018", "US_Post2022", "US_Post2023"]
PERIOD_TERMS = ["US_Period_2018_2021", "US_Period_2022", "US_Period_2023_2024"]
```

#### 核心回归函数

```python
def run_stage_model(
    df, dependent, formula_suffix, model_name, sample_label
):
    # 构造 R 风格公式
    formula = f"{dependent} ~ {' + '.join(STAGE_TERMS)} + {formula_suffix}"
    # f"{dependent} ~ {' + '.join(STAGE_TERMS)} + {formula_suffix}"
    # 展开后类似：
    # "ln_import_value ~ US_Post2018 + US_Post2022 + US_Post2023 + C(exporter_code) + C(product_code) + C(year)"

    model = smf.ols(formula=formula, data=df).fit(cov_type="HC1")
    # smf.ols(...)：普通最小二乘法回归
    # .fit(cov_type="HC1")：使用 HC1 稳健标准误（异方差一致标准误）
    #   → 即使残差的方差不是常数，标准误仍然可信

    rows = _extract_terms(model, STAGE_TERMS, dependent, model_name, sample_label)
    return model, rows
```

**R 风格公式详解**：

```
"ln_import_value ~ US_Post2018 + US_Post2022 + US_Post2023 + C(exporter_code) + C(product_code) + C(year)"
       ↑                    ↑                                  ↑
   被解释变量          核心解释变量（交互项）                 固定效应（控制变量）
```

- `~`：分隔符，左边是被解释变量，右边是解释变量
- `C(exporter_code)`：把 `exporter_code` 当做**分类变量**（categorical variable），为每个值创建一个虚拟变量（dummy）。这就是"出口国固定效应"——每个出口国有一个独立的截距项，吸收了所有不随时间变化的出口国特征（地理距离、语言、制度等）
- `C(year)`：年份固定效应，吸收了宏观经济周期、全球半导体需求等共同时间趋势

这个公式等价于下面这个 C++ 伪代码中的线性模型：

```cpp
// 伪代码：OLS 回归在做什么
double predict(double exporter_fe[200], double product_fe[4], double year_fe[17],
               bool is_us, bool post2018, bool post2022, bool post2023) {
    double result = 0;
    result += exporter_fe[exporter_code];    // 出口国固定效应
    result += product_fe[product_code];       // 产品固定效应
    result += year_fe[year];                  // 年份固定效应
    result += beta1 * (is_us && post2018);   // US_Post2018 的效应
    result += beta2 * (is_us && post2022);   // US_Post2022 的效应
    result += beta3 * (is_us && post2023);   // US_Post2023 的效应
    return result;
}
// OLS 做的事情：找到 beta1, beta2, beta3 和所有固定效应系数，
// 使得预测值和真实 ln_import_value 之间的平方误差最小
```

#### 如何提取回归结果

```python
def _extract_terms(model, terms, dependent, model_name, sample_label):
    rows = []
    conf = model.conf_int()  # 获取所有系数的置信区间

    for term in terms:
        if term not in model.params.index:
            continue  # 如果这个变量不在模型中，跳过

        coef = float(model.params[term])  # 系数估计值（β̂）
        ci_low, ci_high = conf.loc[term].tolist()  # 95% 置信区间

        rows.append({
            "model": model_name,
            "sample": sample_label,
            "dependent_variable": dependent,
            "term": term,
            "coef": round(coef, 6),          # 系数估计值
            "std_error": round(float(model.bse[term]), 6),  # 标准误
            "p_value": round(float(model.pvalues[term]), 6), # p 值
            "ci_low": round(float(ci_low), 6),   # 置信区间下界
            "ci_high": round(float(ci_high), 6), # 置信区间上界
            "nobs": int(model.nobs),             # 样本量
            "r_squared": round(float(model.rsquared), 6),  # R²
            "approx_pct_effect": round(float(np.exp(coef) - 1.0), 6)
            if dependent == "ln_import_value"
            else "",
            # 对 ln 模型：系数近似等于百分比变化
            # β=0.1 → exp(0.1)-1 ≈ 10.5% 增加
        })

    return rows
```

#### 回归结果表各列的含义

| 列名 | 含义 | 类比 |
|------|------|------|
| `coef` | 系数估计值 β̂ | 回归的核心输出。β = 0.1 表示 US_Post2018=1 时 ln(import) 高 0.1 |
| `std_error` | 标准误 | 系数估计的精度。越小越精确 |
| `p_value` | p 值 | 假设检验的概率。p < 0.05 → "显著"（不太可能由随机噪声产生） |
| `ci_low/ci_high` | 95% 置信区间 | 真实 β 有 95% 概率落在这个区间内 |
| `nobs` | 样本量 | 回归用了多少行数据 |
| `r_squared` | R²（拟合优度） | 模型解释了多少变异。0.8 = 80% |
| `approx_pct_effect` | 近似百分比效应 | 仅对 ln 模型有意义 |

```python
    save_table(test_results, "test_results_v02", config)

    # [6/6] 报告
    write_summary(...)
    write_abstract(...)

    print("Completed v0.2 multi-product analysis.", flush=True)
    return 0  # Unix 惯例：返回 0 表示成功
```

这个函数的逻辑就是 README 里描述的 6 步流水线。每一步输出作为下一步的输入，形成了清晰的数据依赖链。

**为什么 `main()` 返回 `int` 而不是 `void`？**

Python 通常不要求 main 返回值，但这里显式返回 `0` 是为了和 Unix 进程返回码惯例保持一致。入口脚本中用 `raise SystemExit(main())` 把这个 int 转为进程退出码，这样 `echo $?` 可以检查运行是否成功。

## 7. Python 环境概念补充

### 7.1 虚拟环境（Virtual Environment）

```bash
python3 -m venv .venv
```

创建一个独立的 Python 环境。类比：给这个项目一个独立的环境，不影响系统 Python 和其他项目的依赖。类似 C++ 中的 `-I/path/to/project/libs` 但没有全局依赖冲突问题。

### 7.2 pip（Package Installer for Python）

```bash
pip install pandas
```

pip 就是 Python 的包管理器，类似 C++ 的 vcpkg/conan 或 Linux 的 apt。从 PyPI（Python Package Index）下载包并安装到当前虚拟环境。

### 7.3 requirements.txt

```
pandas==3.0.2
numpy==2.4.4
matplotlib==3.10.8
statsmodels==0.14.6
```

锁定具体版本号。`pip install -r requirements.txt` 会安装这些精确版本，保证**可复现性**——别人（或一年后的你）跑这个项目时，依赖版本和开发时完全一致。

### 7.4 `__init__.py`

`scripts/v02_analysis/__init__.py` 是一个空文件。它的存在告诉 Python："这个目录是一个**包（package）**"，可以被 `import v02_analysis.config` 这样的语句找到。没有这个文件的目录不能作为包导入。Python 3.3+ 支持隐式命名空间包，但显式加 `__init__.py` 是更安全、更明确的做法。
