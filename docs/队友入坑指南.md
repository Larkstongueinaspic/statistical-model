## 1. 这个项目是干什么的

### 一句话

研究 **2018 年以来的美国出口管制政策** 是否改变了 **中国从美国进口半导体相关产品的贸易格局**。

### 背景

2018 年起美国逐步升级对华半导体出口管制——实体清单、芯片法案、设备禁令层层加码。我们用 CEPII BACI 全球贸易数据库（HS6 产品级，2008-2024 年），量化这些政策对中国进口来源结构的影响。

### 研究的产品

| HS6 编码 | 含义 | 类别 |
|----------|------|------|
| 848620 | 半导体器件或集成电路制造设备 | 设备类 |
| 854231 | 处理器及控制器（CPU/GPU） | 集成电路类 |
| 854232 | 存储器（DRAM/NAND） | 集成电路类 |
| 854239 | 其他电子集成电路 | 集成电路类 |

### 核心发现

- **美国份额在下降**：848620 的美国份额从 2017 年 27.92% 降到 2024 年 9.88%。854231、854232 也在降。但 854239 反而小涨——半导体产品不是铁板一块。
- **替代来源不同**：设备类转向日本、荷兰、新加坡；集成电路类更多依赖韩国、马来西亚、越南等亚洲生产网络。
- **回归不显著**：固定效应模型里 US_Post2018/2022/2023 都不显著。**不能写成"美国出口管制导致对华出口显著下降"**——只能说"来源结构重组，美国份额承压"。

## 2. 最近做了什么（v0.2 + 论文）

### 新增了 SIRI 风险指数

SIRI = Semiconductor Import Risk Index，半导体进口供应链风险指数。

从四个维度按产品-年份打分（0-100，越高越危险）：

- **来源集中风险**：进口来源国越集中，风险越高
- **政策暴露风险**：来自美国的份额越大，在管制背景下暴露越高  
- **替代不足风险**：非美国来源也集中在少数国家，替代弹性差
- **结构波动风险**：相邻年份来源份额变化越大，供应链越不稳定

基准权重各 25%。还做了一个敏感性版本（政策暴露权重提到 40%）来验证排序是否稳定。

2024 年 SIRI 排名：854239 > 854232 > 848620 > 854231。

### 写了一篇 LaTeX 论文草稿

`paper/main.pdf` —— 12 页，可以编译成 PDF。

内容包括：摘要、引言、数据说明、描述统计、SIRI 风险指数、政策节点回归（固定效应 + 稳健性检验）、结论与政策建议、附录。

**正文约 5500 字，竞赛限制 16000 字，还有充足空间可扩展。**

## 4. 怎么把环境搭起来、跑一遍

### 4.1 前提条件

- Python 3.9+（推荐 3.11+）
- 约 20GB 磁盘空间（BACI 原始数据）
- 约 8GB 可用内存

### 4.2 下载数据

```bash
cd statistical-model
wget https://www.cepii.fr/DATA_DOWNLOAD/baci/data/BACI_HS07_V202601.zip
unzip BACI_HS07_V202601.zip
```

### 4.3 创建虚拟环境 + 装依赖

```bash
python3 -m venv .venv
source .venv/bin/activate        # Linux/Mac
# 或 .venv\Scripts\activate     # Windows
pip install -r requirements.txt
```

### 4.4 跑完整分析流水线

```bash
.venv/bin/python scripts/run_v02_analysis.py
```

跑完后去 `results/v02/` 看图表和表格，去 `docs/output/` 看自动生成的 summary。

### 4.5 编译论文 PDF

论文用 XeLaTeX 编译（需要装 TeX Live 2026+）。

Arch Linux 需要装这些包：
```bash
sudo pacman -S texlive-bin texlive-basic texlive-xetex texlive-latex texlive-latexextra \
  texlive-latexrecommended texlive-bibtexextra texlive-fontsrecommended \
  texlive-langchinese texlive-fontsextra
```

其他系统请装完整的 TeX Live（含 xetex 和 ctex 包）。

然后编译：
```bash
cd paper
xelatex main.tex       # 第一遍
bibtex main             # 处理参考文献
xelatex main.tex       # 第二遍
xelatex main.tex       # 第三遍（交叉引用稳定）
```

生成 `paper/main.pdf` 即最终论文。

如果不想装 TeX Live，也可以用 Overleaf——把整个 `paper/` 目录上传，编译器选 XeLaTeX。

### 4.6 跑单元测试

```bash
.venv/bin/python -m unittest tests.test_v02_risk_index -v
```

应该显示 7 个 test 全 PASS。

## 7. 常见问题

**Q: 我不想装 Python，能看结果吗？**

能。`results/v02/tables/` 里的 `.md` 文件用任何文本编辑器都能打开看。`results/v02/figures/` 里的 `.png` 图表直接双击就能看。

**Q: BACI 数据要钱吗？**

不要。CEPII BACI 对学术研究免费开放，但需要在 CEPII 网站注册账号才能下载。

**Q: 我能把产品换成其他的吗？**

能。改 `scripts/v02_analysis/config.py` 里的 `candidate_product_codes` tuple，然后重新跑流水线。但要确认新产品的 HS6 编码在 BACI 数据库里有足够数据。

**Q: 论文用的什么模板？**

自建的 XeLaTeX 模板（`ctexart` 文档类），不是官方比赛模板——因为我们确实没找到官方模板。如果有官方模板，需要把 `paper/sections/` 里的内容迁移过去。

**Q: 编译报错了怎么办？**

1. 确认装了 `texlive-xetex` 和 `texlive-langchinese`
2. 确认装了 `texlive-fontsrecommended`（拉丁现代字体）
3. 确认运行了三次 `xelatex`（解决交叉引用）
4. 如果还报错，把错误信息贴给我
