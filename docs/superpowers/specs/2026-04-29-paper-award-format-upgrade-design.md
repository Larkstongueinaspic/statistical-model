# 论文获奖格式对齐与冲奖补强设计

日期：2026-04-29

## 1. 背景与目标

项目当前已经形成 `paper/main.tex`、分章节 `paper/sections/*.tex`、论文图表和 `paper/main.pdf`。新增参考材料包括：

- `docs/模板.tex`：推荐模板，已包含中文论文基础结构、页眉页脚、摘要装饰线和章节骨架。
- `docs/获奖论文节选.pdf`：获奖论文集节选，PDF 为扫描图像，不能可靠抽取正文文本，但可以通过页面渲染观察版式。

本轮目标是采用方案 C：将 `docs/模板.tex` 升级为可复用母版，同时让 `paper/main.tex` 和正文内容同步套用这套风格，并补强论文的竞赛表达，使论文更接近全国大学生统计建模大赛获奖论文的呈现方式。

目标不是把最终论文做成扫描书稿或横向双页 PDF，而是在单页 A4 提交论文中吸收获奖论文的单页版式、中文编号层级、图表规范、建模展示方式和竞赛论文叙事结构。

## 2. 获奖论文格式观察

`docs/获奖论文节选.pdf` 是获奖论文集扫描件，每个 PDF 页面多为书本左右双页。因此页面本身的横向尺寸不应被直接模仿。可借鉴的是单篇论文在单页内的排版习惯：

- 首页采用居中大标题，作者和单位居中，摘要标题居中，摘要正文紧凑，关键词位于摘要末尾。
- 正文层级以中文编号为主，常见结构为 `一、`、`（一）`、`1.`、`（1）`。
- 页眉包含论文题名或论文集标题，并有顶部横线；页码位于页脚外侧或底部。
- 正文段落密度高，段前段后空白少，图表与正文衔接紧凑。
- 图题、表题居中，编号格式为“图 1 ...”“表 2 ...”。
- 流程图、模型框架图、指标体系图和评价结果图是常见加分展示。
- 参考文献采用顺序编码 `[1]`、`[2]` 风格，而不是作者年份风格。

当前 `paper/main.pdf` 更接近普通 LaTeX 论文风格：章节编号、摘要页、页眉页脚、参考文献和正文密度都还没有完全对齐竞赛获奖论文气质。

## 3. 竞争力差距

### 3.1 格式差距

当前论文已经可编译，但需要向获奖论文风格靠近：

- 将章节编号从阿拉伯数字标题改为中文竞赛论文层级。
- 统一摘要、目录、正文、附录、参考文献的页眉页脚和页码风格。
- 压缩正文段距和浮动体间距，提高单位页信息密度。
- 将图表标题、注释和表格线型统一为竞赛论文风格。
- 将参考文献切换为顺序编号引用。

### 3.2 内容差距

当前论文的核心结果稳健，但竞赛表达还偏克制：

- 缺少独立文献综述，无法充分展示研究基础和问题定位。
- 缺少总研究技术路线图，评委需要从文字中自行拼接研究流程。
- SIRI 指数有公式和排序，但缺少指标体系图、分维度展示和风险分层表达。
- v0.3 预测扩展已经存在，但需要更清楚地定位为“预警框架原型”，避免弱化主线或夸大模型效果。
- 政策建议可以更落地，按产品类别、风险来源和监测机制展开。

### 3.3 需要保持的优势

论文当前的优势不能被过度包装破坏：

- 数据口径清楚，基于 BACI HS07 2008--2024 年中国进口产品级数据。
- 结论边界清楚，不把固定效应回归不显著硬写成强因果结果。
- SIRI 指标可解释、可复现，能连接描述事实和风险评价。
- v0.3 已形成 20 个 HS6 产品、年度贸易网络和下一年 SIRI 预测任务，适合作为扩展展示。

## 4. 推荐方案

采用“C 的精简冲奖版”：

1. `docs/模板.tex` 升级为母版，沉淀可复用格式。
2. `paper/main.tex` 同步套用母版风格，保持当前 `paper/sections/` 分文件结构。
3. 补强内容只围绕主线展开，不堆砌无关模型。
4. 新增或强化三类展示：研究技术路线图、SIRI 指标体系与风险分层、文献综述与创新点。
5. v0.3 保留为“预测与预警扩展”，不声称 GCN 显著优于基线，不声称已经完成真实 GDELT 建模。

## 5. 设计范围

### 5.1 需要修改的文件

- `docs/模板.tex`：升级为获奖论文风格母版。
- `paper/main.tex`：套用同一套格式设置，并调整参考文献、页眉页脚、标题和章节样式。
- `paper/sections/abstract.tex`：从 LaTeX `abstract` 环境改为与母版一致的摘要块或配套命令。
- `paper/sections/introduction.tex`：拆分和扩写绪论、研究内容与创新点。
- `paper/sections/literature_review.tex`：新增文献综述与理论基础章节，避免把绪论写得过长。
- `paper/sections/data.tex`：增强数据口径和变量构造说明。
- `paper/sections/descriptive.tex`：按获奖论文层级重排描述统计与结构演化分析。
- `paper/sections/risk_index.tex`：强化 SIRI 指标体系、分维度解释和风险分层。
- `paper/sections/regression.tex`：保留统计边界讨论，增强“描述事实与回归证据关系”的表达。
- `paper/sections/prediction_extension.tex`：明确 BACI-only 预测扩展定位。
- `paper/sections/conclusion.tex`：将政策建议拆成更落地的分点。
- `paper/references.bib`：补充必要参考文献条目，并改用顺序编码引用样式。

### 5.2 可新增的文件

允许新增由 LaTeX 或脚本生成的论文图表资源，但必须服务主线。优先级如下：

1. 总研究技术路线图。
2. SIRI 指标体系图。
3. 2024 产品风险分层或分维度展示图。

新增图表若能直接用 LaTeX/TikZ 或现有数据生成，优先采用可复现方式。若时间受限，也可以先用 LaTeX 原生 `tabular`、`fbox`、`minipage` 形成简洁框图。

### 5.3 不纳入本轮

- 不将 PDF 做成扫描书稿效果。
- 不改动现有分析数据结果，除非后续实施计划明确需要重跑图表。
- 不引入新的复杂因果识别设计。
- 不声称真实 GDELT 已接入。
- 不声称贸易网络原型或 GCN 显著优于所有基线。

## 6. 论文结构设计

最终正文建议采用如下结构：

```text
摘要
关键词
目录

一、绪论
  （一）研究背景与问题提出
  （二）研究意义
  （三）研究内容与技术路线
  （四）主要创新点

二、文献综述与理论基础
  （一）供应链风险测度研究
  （二）半导体贸易与出口管制研究
  （三）贸易网络与风险预测研究
  （四）文献评述与本文定位

三、数据来源与变量构造
  （一）数据来源与样本范围
  （二）产品选择与样本构造
  （三）变量定义与模型设定

四、半导体进口来源结构演化分析
  （一）总进口规模变化
  （二）美国来源份额变化
  （三）主要来源国结构
  （四）来源集中度与替代路径

五、SIRI 供应链风险指数构建与评估
  （一）指标体系设计
  （二）指数计算方法
  （三）产品风险排序与分层
  （四）权重敏感性检验

六、政策节点下的统计检验与证据边界
  （一）固定效应模型设定
  （二）回归结果解释
  （三）稳健性分析
  （四）统计边界讨论

七、预测与预警扩展：基于贸易网络的 SIRI 风险预测
  （一）扩展产品池与年度贸易网络
  （二）下一年 SIRI 预测任务
  （三）模型比较与结果解释
  （四）扩展价值与局限

八、结论与政策建议
参考文献
附录
```

该结构比当前版本更像获奖论文，同时保留现有主线，不将预测扩展喧宾夺主。

## 7. 模板设计

### 7.1 页面与页眉页脚

- 保持 A4 纵向页面。
- 使用 `ctexart` + XeLaTeX。
- 本轮保持 `docs/模板.tex` 为可独立编译或阅读的母版；`paper/main.tex` 镜像采用同一套关键格式设置。暂不抽取共享 `.sty` 文件，避免相对路径和提交材料迁移带来额外复杂性。
- 页边距维持约 `left=2.8cm,right=2.8cm,top=2.6cm,bottom=2.6cm`，可在实施时微调以提高正文密度。
- 正文页启用 `fancyhdr`：页眉中间显示短论文题名或赛事标题，页眉横线宽度约 `0.4pt`。
- 封面页不显示页眉页脚。
- 页码放在底部，优先保持居中；若实现外侧页码不会引入复杂性，可改为奇偶页外侧。

### 7.2 标题层级

- `\section` 显示为 `一、标题`。
- `\subsection` 显示为 `（一）标题`。
- `\subsubsection` 显示为 `1. 标题`。
- 正文内列表尽量采用紧凑 `enumerate` 或直接段落，不使用松散项目符号堆叠。

### 7.3 摘要与关键词

- 摘要标题居中加粗。
- 摘要正文不用默认 `abstract` 环境的大缩进样式，改为正文宽度内紧凑排版。
- 关键词以 `\textbf{关键词：}` 开头，置于摘要正文后。

### 7.4 图表

- 图表标题居中，使用“图 1 ...”“表 1 ...”。
- 表格优先使用 `booktabs`，大型表格使用 `longtable`。
- 图表注释使用 `\caption*{注：...}` 或统一命令，字号小于正文。
- 避免大块空白。必要时调整图片宽度、浮动位置和上下间距。

### 7.5 参考文献

- 改用顺序编码样式，例如 `unsrtnat` 或 `plainnat` 的数字模式。
- 正文引用从作者年份风格转为 `[1]` 风格。
- 补充文献时优先选择权威来源和与研究直接相关的文献，避免堆参考文献。

## 8. 内容增强设计

### 8.1 文献综述

新增独立章节“文献综述与理论基础”。写作重点：

- 供应链风险测度：说明单一来源依赖、集中度、韧性与替代能力等维度。
- 半导体贸易与出口管制：说明出口管制是重要政策背景，但贸易结构变化受多因素影响。
- 贸易网络与风险预测：说明网络结构可以服务风险监测，但预测结果不能替代因果识别。
- 文献评述：指出本文的定位是产品级贸易数据、可解释风险指数和谨慎统计推断。

### 8.2 研究技术路线图

在绪论中新增总体技术路线图，展示：

```text
BACI 数据 → 产品筛选 → 来源结构描述 → SIRI 风险指数 → 固定效应检验 → 预测扩展 → 政策建议
```

该图应简洁，不需要复杂视觉效果。目标是让评委快速理解论文工作量和逻辑链条。

### 8.3 SIRI 指标体系图

在 SIRI 章节新增指标体系图：

```text
SIRI
├─ 来源集中风险：HHI
├─ 政策暴露风险：美国份额
├─ 替代不足风险：非美来源内部 HHI
└─ 结构波动风险：相邻年份总变差距离
```

该图用于把公式转化为可读的评价体系，增强竞赛论文表达。

### 8.4 风险分层表达

补强 2024 年产品风险排序，形成“高风险、中风险、低风险”的解释：

- 高风险：854239、854232，主因是来源集中和替代不足。
- 中风险：848620，政策暴露较高但来源多极。
- 低风险：854231，来源更均衡。

风险分层必须与现有数据一致，不引入新结论。
实施计划中需要安排一次对 `paper/tables/siri_ranking_2024.tex` 和相关结果表的核对，确认分层表述与 2024 年 SIRI 分数、美国份额、HHI 和替代不足指标一致。

### 8.5 v0.3 表述边界

预测扩展章节保留，但需强调：

- 当前为 BACI-only 贸易网络预测扩展。
- 20 个 HS6 产品和 320 个年度图样本体现扩展工作量。
- 简单基线在测试集仍有竞争力，复杂贸易网络原型没有稳定优势。
- 该扩展的价值是把 SIRI 推进到下一年风险预测任务，为后续接入 GDELT 留出接口。

## 9. 实施边界与风险控制

### 9.1 编译风险

可能风险：

- 中文章节编号重定义与 `ctexart` 默认标题格式冲突。
- `natbib` 数字引用模式与现有 `\citep` 命令不兼容或显示不理想。
- 图表浮动压缩后出现过满盒或大块空白。

控制方式：

- 小步修改 `paper/main.tex`，每轮编译确认错误位置。
- 保留现有分文件结构，避免一次性合并全部章节。
- 使用 `xelatex`、`bibtex`、`xelatex`、`xelatex` 完整编译链验证交叉引用。

### 9.2 内容风险

可能风险：

- 补强后篇幅过长，接近竞赛字符上限。
- 文献综述变成泛泛堆砌，与本文实证主线脱节。
- v0.3 被写得过强，削弱论文诚实边界。

控制方式：

- 文献综述控制在竞赛论文需要的长度，只服务本文定位。
- 新增图表最多优先做三类，不堆无关展示。
- 所有预测扩展表述继续标注 BACI-only 和探索性。

## 10. 验证标准

实施完成后需要满足：

1. `docs/模板.tex` 能作为后续复用母版阅读和编译参考。
2. `paper/main.tex` 能完整编译生成 `paper/main.pdf`。
3. PDF 版式明显接近获奖论文风格：中文编号、紧凑正文、规范页眉页脚、图表居中编号、顺序编号参考文献。
4. 新增内容不改变已有实证结论，不夸大政策因果效应或预测模型效果。
5. 论文主线清楚：来源结构演化 → SIRI 风险指数 → 统计检验边界 → 预测预警扩展。
6. 工作区不混入无关结果文件改动。
7. 最终 PDF 仍为 A4 纵向单页论文形态，不模仿获奖论文集扫描件的横向双页书稿形态。

## 11. 成功标准

本轮完成后，论文应达到以下状态：

- 读者第一眼能看出这是统计建模竞赛论文，而不是普通课程论文草稿。
- 评委能快速识别本文的研究对象、数据口径、建模框架、核心发现和创新点。
- SIRI 指数成为论文最稳定的核心模型亮点。
- v0.3 预测扩展成为工作量和可扩展性的加分项，而不是风险表述点。
- 模板文件和论文文件保持一致，后续只需补队伍信息和最终润色即可提交。
